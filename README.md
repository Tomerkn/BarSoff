# מערכת Barsuf - פלטפורמה לניהול פרויקטים ומכרזים

מערכת Barsuf היא פלטפורמה אחודה לניהול פרויקטים, פיקוח הנדסי, בקרה תקציבית וניתוח מכרזים חכם מבוסס בינה מלאכותית. המערכת מאפשרת למנהלי פרויקטים ולצוותים בשטח לנהל את כל היבטי הבנייה והביצוע תחת ממשק אחד מהיר ומאובטח.

## מודולים מרכזיים במערכת

1. **ניהול מכרזים חכם (ברבור)**: מנגנון ניתוח מתקדם המשלב עיבוד דו-שלבי לקבלת תובנות מיידיות ממסמכי מכרז בתוך שניות ספורות (כולל חילוץ כתב כמויות ראשוני ומחשבון מחיר מטרה דינמי מבוסס סליידרים), ולאחר מכן ביצוע ניתוח עומק והפקת הצעות מחיר מפורטות.
2. **ניהול לוחות זמנים (גאנט)**: תצוגת WBS וגאנט אינטראקטיבית למעקב אחר שלבי הביצוע והמשימות בפרויקט, כולל תמיכה מלאה בסנכרון דו-כיווני מול מערכת Monday.com.
3. **בקרה תקציבית ופיננסית**: מודולים מובנים לניהול תקציב היעד, רישום הוצאות בפועל, רישום הכנסות וניהול תשלומים לקבלנים.
4. **ניהול קבלני משנה**: רישום ועריכה של קבלני משנה והתמחויותיהם.
5. **יומני עבודה יומיים**: כלי לדיווח יומיומי של מנהלי העבודה בשטח כולל תיאור ביצוע, הערכות מפקח ורישום תקלות.
6. **שנת בדק**: ניהול קריאות שירות, תקלות בתקופת הבדק ומעקב אחר סטטוס הטיפול מול דיירים וקבלנים.
7. **דוחות ואנליטיקה**: הפקת דוחות רווח והפסד, השוואת תקציב מול ביצוע וניתוח מדדים כספיים ברמת הפרויקט והחברה.

## ארכיטקטורה וטכנולוגיות

המערכת בנויה על גבי תשתית מודרנית המשלבת ביצועים גבוהים ופשטות תפעולית:

* **צד לקוח (Frontend)**:
  * React 18: ספריית התצוגה המרכזית.
  * Vite: כלי הבנייה וההרצה המהיר.
  * Tailwind CSS: תשתית העיצוב (סטייל מותאם אישית ללא תלות ברכיבים חיצוניים כבדים).
  * Lucide React: חבילת אייקונים וקטוריים מינימליסטיים.
  * React Router DOM: ניהול הניווט ומצבי הדפים.

* **צד שרת (Backend)**:
  * Node.js עם Express: שרת ה-API המרכזי המנהל את הלוגיקה העסקית.
  * SQLite (Better-SQLite3): מסד הנתונים הראשי למען ביצועים גבוהים ופשטות הרצה.
  * Multer: ניהול העלאות קבצים בצורה מאובטחת.
  * pdf-parse: מנוע המרת קבצי PDF לטקסט לצורך ניתוח מהיר של המכרזים.

* **בינה מלאכותית (AI)**:
  * Anthropic Claude API (Claude 3.5 Sonnet ו-Claude 3 Haiku): משמש כמנוע הראשי לניתוח מהיר וניתוח עומק של מסמכים וכתבי כמויות במכרזים.
  * Google Gemini API (Gemini 1.5 Pro / Flash): משמש כגיבוי ומנוע לצ'אט החכם של הבוט הצף בפרויקטים.
  * מנגנון וקטורי מובנה: לטובת שליפת נתוני עבר והיסטוריית מחירי שוק לשם תמחור מדויק.

* **ניהול נתונים וגיבוי**:
  * Google Cloud Storage: סנכרון וגיבוי אוטומטי של קובץ ה-SQLite ישירות לענן בכל 5 דקות כדי למנוע אובדן מידע בסביבות Cloud Run.

## זרימת ניתוח המכרזים (ברבור)

כאשר משתמש מעלה מכרז חדש:
1. המערכת שומרת את הקובץ ומחזירה תגובה מיידית בתוך כחצי שנייה.
2. ברקע מופעל מנוע הניתוח הדו-שלבי:
   * **שלב 1 (מהיר)**: שימוש ב-Claude Haiku המנתח את 4,000 התווים הראשונים ומחזיר סיכום נקודות מפתח בתוך כ-3-4 שניות. הסטטוס מתעדכן ל-"נותח (ראשוני)" והמשתמש מקבל תצוגה ראשונית.
   * **שלב 2 (עמוק)**: שימוש ב-Claude Sonnet המבצע ניתוח מקיף של כלל סעיפי המכרז (עד 15,000 תווים), מחלץ כתב כמויות (BoQ) בפורמט JSON ומעדכן את הממשק אוטומטית. מחשבון מחיר המטרה מוצג למשתמש באופן מיידי.

## איך מפעילים את הפרויקט מקומית

1. **התקנת תלויות**:
   מריצים בתיקייה הראשית:
   `npm install`

2. **קובץ הגדרות סביבה (.env)**:
   יש ליצור קובץ `.env` בתיקיית השרת או בתיקייה הראשית ולהגדיר את הפרמטרים הבאים:
   * `PORT`: פורט להרצת השרת (ברירת מחדל 8080).
   * `ANTHROPIC_API_KEY`: מפתח ה-API של Anthropic לטובת הפעלת Claude.
   * `GEMINI_API_KEY`: מפתח ה-API של Google לטובת Gemini.

3. **הרצה במצב פיתוח**:
   מריצים מהתיקייה הראשית:
   `npm run dev`
   הפקודה תפעיל במקביל את שרת ה-API (פורט 3001) ואת ממשק ה-Vite של צד הלקוח (פורט 5173).

4. **בנייה לשרת ייצור**:
   לצורך קימפול ואיחוד הקבצים:
   `npm run build`
   הקבצים המוכנים יישמרו בתיקיית `dist` ויוגשו סטטית על ידי שרת ה-Express.
